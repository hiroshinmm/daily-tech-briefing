# 実装計画: 動的画像取得の安定化（全スライド同一画像問題の修正）

## 概要
動的画像取得において、すべてのスライドに同じ画像が表示される問題を修正します。LoremFlickr の仕様に合わせ、検索クエリの最適化とキャッシュ回避策を導入します。

## 修正内容

### 1. `src/captureSlides.js` の修正
- **タグのクリーニング**: カンマやスペースによる重複を排除し、綺麗なタグリストを作成します。
- **タグ数の制限**: 関連性の高い上位 3 つのタグのみを使用するように制限します（多すぎるとヒットしません）。
- **検索ロジックの緩和**: `/all` オプションを削除し、いずれかのタグにマッチすれば表示されるようにします（ヒット率向上）。
- **ランダムシードの追加**: 各リクエストに異なる `random` パラメータを付与し、同一キーワードでも異なる画像が選ばれやすく、かつキャッシュされないようにします。

```javascript
// 修正後のロジック案
const tagList = keywords.split(/[,\s]+/).filter(t => t.length > 1).slice(0, 3);
const queryTags = tagList.join(',').toLowerCase();
const imageUrl = `https://loremflickr.com/1920/1080/${encodeURIComponent(queryTags)}?lock=${Math.floor(Math.random() * 1000)}`;
```

## 検証プラン
### 自動テスト
- `captureSlides.js` を実行し、各スライドの `[DEBUG]` ログで生成された URL が異なっていること、および出力された画像が別々であることを確認します。

### 手動確認
- 生成された `dist/images/` フォルダを直接開き、すべての画像が異なることを目視で確認します。
